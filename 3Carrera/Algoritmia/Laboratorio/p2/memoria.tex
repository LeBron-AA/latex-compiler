\result{Tiempo teórico de ordenación de un vector}
\hspace{3mm}
En el problema propuesto, el vector está ordenado aleatoriamente. Por tanto, hemos de usar la complejidad teórica correspondiente al caso medio para cada algoritmo. Esta es de $O(n^2)$ para todos los algoritmos vistos; a excepción de \textit{QuickSort}, que tiene una complejidad de $O(n \log n)$ para este caso.

\vspace{4mm}
Sea $f_i$ la función de complejidad teórica para el algoritmo teórico $i$-ésimo, conocido el tiempo $t_1$ que emplea ese mismo algoritmo para una carga $n_1$, podemos predecir el tiempo para otra carga $n_2$ como:
$$ t_2 = t_1 \hspace{1mm} \cdot \frac{f_i(n_2)}{f_i(n_1)}$$

Intentaremos resolver este problema teórico usando el tiempo de mayor carga medido para cada algoritmo. Al usar el caso de mayor $n$ estudiado, las variaciones que pueden ocurrir al llamar a una función ajenas al algoritmo de ordenación se ven mitigadas: El coste fijo del problema (ajeno al tamaño de la carga) se vuelve despreciable.

\vspace{2mm}
\begin{itemize}
  \item \textbf{Burbuja:} 56.764ms para una repetición en el caso $n=80.000$.
  $$56.764ms \cdot \frac{81.920.000^2}{80.000^2}\cdot \frac{1d}{86.400.00 ms} = 688,9047d$$
  \newpage
  \item \textbf{Selección:} 80.194ms para el caso $n=160.000$
  $$80.194ms \cdot \frac{81.920.000^2}{160.000^2}\cdot \frac{1d}{86.400.00 ms} = 243,3145d$$

  \vspace{4mm}
  \item \textbf{Inserción:} 174.247ms para el caso $n=320.000$
  $$174.247ms \cdot \frac{81.920.000^2}{320.000^2}\cdot \frac{1d}{86.400.00 ms} = 132,1696d$$

  \vspace{4mm}
  \item \textbf{\textit{QuickSort:}} 71.247ms para el caso $n=1.280.000$ en 100 repeticiones.
  $$\frac{71.247ms}{100} \cdot \frac{81.920.000 \cdot \log(81.920.000)}{1.280.000 \cdot \log(1.280.000)}\cdot = 59,0835ms$$
  En días, son $6,8384 \cdot 10^{-7}d$.

\end{itemize}

\result{Mediciones con tiempos bajos}
\hspace{3mm}
En primer lugar, hemos medido los tiempos de Inserción y \textit{QuickSort} para cargas bajas. Para ello, hemos tenido en cuenta el tiempo del bucle al completo, en lugar de ir sumando los tiempos de cada paso. De este modo, no tenemos problemas con hipotéticas medidas nulas.

\vspace{4mm}
Aunque \textit{QuickSort} maneja mejor problemas grades, es más lento que Inserción para cargas bajas. Tras medir estos dos algoritmos empíricamente, hemos diseñado un tercero que usa el más rápido según el tamaño del problema: Divide la carga con \textit{QuickSort} hasta llegar a un tamaño en el que Inserción es más rápido, en cuyo caso llama a este último método.
